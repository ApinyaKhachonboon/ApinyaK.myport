GIT

หนึ่งใน Version Control เป็นที่นิยมในการทำงานรวมกันของ Programmer และ Developer ทำให้จัดการ code ได้ง่ายเป็นระเบียบนอกจากนี้ยังสามารถใช้ git เป็น web server ได้อีกด้วย โดยการทำงานแยกเป็น 7 ข้อดังนี้

1.เก็บประวัติ

การแก้ไข

- ใครเป็นคนแก้

- แก้ตรงไหนบ้าง

- ของเดิมเป็นอย่างไร

2.รวม code

3.สามารถตามดูประวัติได้

4.Systematic มีระบบระเบียบในการจัดการ

5.Backup

6.Rollback

7.Track

จะมี Server ซึ่งเรียกว่า Remote และ Repository หรือ folder ของ project นั้นเอง โดยจะมีขั้นตอนหลักตามนี้คือ

1.Clone -> copy Repo จาก Remote

2.Commit -> save local เตรียมที่จะ upload ขึ้น Remote

3.Unstash -> กรณีมี branch แยกจาก project หลัก

4.stash -> กล่องเก็บ code ที่ยังไม่ commit

5.Push -> upload code ขึ้น Remote

6.fetch -> Refresh ข้อมูลก่อนดึงมา

7.Pull -> ดึงข้อมูลจาก Remote มา

8.Conflict -> ในกรณีมี conflict ให้แก้ก่อน

9.merge -> รวม code กับ project หลัก

การสั่งการทำงานของ GIT ทำได้หลายรูปแบบ เช่นใช้ source tree, git desktop หรือ Git bash ในกรณีเลือกใช้ Git bash จะใช้คำสั่งในการทำงานหลักๆ คือ

1.git config –global user.name -> ใส่ username ของผู้ใช้

2.git config –global user.email -> ใส่ email ของผู้ใช้

3.git clone ตามด้วย url ของ repo ที่ต้องการจะ clone

4.git status -> เช็คว่ามีการแก้ไขไฟล์ไหนใน repo หรือไม่

5.git add ตามด้วยชื่อไฟล์ หรือ “.” แทนทุกไฟล์

6.git commit -m ตามด้วยคำอธิบาย

7.git push origin ตามด้วยชื่อ branch ที่ต้องการ upload
