Java

การเขียนโปรแกรมในรูปแบบของ OOP หรือเรียกเต็มๆว่า Object Oriented Programming หรือเรียกว่าการเขียนโปรแกรมเชิงวัตถุ เพื่อให้ง่ายต่อการแก้ไขหรือคงสภาพไว้ จึงจำเป็นต้องมีแบบแผนในการเก็บที่เหมาะสมพร้อมใช้งาน โดยการมองทุกสิ่งเป็นวัตถุทั้งหมด โดยแยกเป็น 2 อย่างหลักๆ 1.คุณสมบัติ (Attributes or properties) 2.พฤติกรรม (Method) เมื่อเอามารวมกันจะกลายเป็น 1 Object ซึ่งจะมีอยู่ 3 รูปแบบหลักๆ คือ

1.Encapsulation การห่อหุ้ม คือ

การที่ข้อมูลหรือคุณลักษณะบางอย่างที่สร้างขึ้นในคลาสสามารถปกป้องหรือถูกห่อหุ้มอยู่ภายในไม่ให้ Object สามารถเข้าใช้งานหรือเข้าไปแก้ไขข้อมูลได้ ตัวอย่างเช่นการสร้างคลาสวงกลมชื่อ Circle โดยภายในคลาสจะมีการ Method สำหรับกำหนดค่ารัศมี Method สำหรับหาพื้นที่วงกลม และ Method หาเส้นรอบวง และมีคุณลักษณะเป็น รัศมีวงกลม และตัวแปร Pi สำหรับใช้เป็นค่าคงที่ในการคำนวณหาพื้นที่และเส้นรอบวงกลม สำหรับค่า Pi นั้นควรใช้ภายในคลาสเท่านั้น ไม่ควรให้ Object เข้าไปแก้ไขได้ เนื่องจากถ้าหากมีการแก้ไขจะทำให้การคำนวณหาค่าเส้นรอบวง และพื้นที่ผิดไปหมด คุณลักษณะแบบนี้เรียกว่า การห่อหุ้ม

2.Inheritance การสืบทอด คือ

การนำ Object มาสร้างลำดับความสัมพันธ์เป็นลำดับขั้น ซึ่งถือเป็นคุณลักษณะเด่นอย่างหนึ่งทำให้สามารถนำคลาสที่มีอยู่แล้ว หรือนำโปรแกรมที่ได้เขียนไว้แล้วนำกลับมาใช้ได้ใหม่อีก ยกตัวอย่างเช่น คลาส Student ที่ใช้เก็บข้อมูลนักเรียนที่เคยสร้างเอาไว้ หากต้องการสร้างคลาสใหม่ให้เก็บคะแนนวิชาฟิสิกส์เพิ่มเติม ซึ่งทำให้ Object ของคลาสใหม่ยังมี Attributes เหมือนเดิม แต่มีคะแนนวิชาฟิสิกส์เพิ่มขึ้นมา

3.Polymorphism การมีได้หลายรูปแบบ คือ

เป็นการยอมให้คลาสทั่วๆไปมี Method การทำงานร่วมกันได้ คุณสมบัตินี้จะทำให้ Object ต่างๆ ที่มีในโปรแกรมสามารถทำงานจากคำสั่งเดียวกันในโปรแกรมได้จากนั้นแต่ละ Object ก็จะไปทำตามรูปแบบที่กำหนดไว้ในตัวเอง ยกตัวอย่างเช่น มีคลาส Interface ชื่อ Shapes ซึ่งจะมีคุณสมบัติคือ ขนาด ตำแหน่ง และมี Method สำหรับวาด ถ้าเราต้องการรูปสามเหลี่ยมก็แค่เรียกใช้ Interface ของ shapes และกำหนดคุณสมบัติของสามเหลี่ยมไป หรือจะเป็นสี่เหลี่ยม ก็ใช้วิธีแบบเดียวกัน
