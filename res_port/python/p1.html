Pac-man

Class Diagram

Class gameState

- เป็นคลาสสำหรับสร้าง canvas จาก tkinter ขึ้นมา และกำการกำหนดค่าเริ่มต้นต่างๆ ขึ้นมา เช่น สร้าง block จาก file txt ซึ่งเป็น map ของเกม ต่อมาสร้าง foodsmall และ foodbig หรือเป็น object ที่ทำหน้าที่เป็นอาหารของตัว player หรือ ตัว pac-man และ สร้าง ghost ที่เป็นศัตรูของ player และสั่งให้ player, ghost, foodbig ทำงานโดยผ่าน method action

Class player

- เป็นคลาสแทนตัวผู้เล่น หรือตัว pac-man นั้นเอง ใน constructor นั้นจะมีค่า ขนาด, ตำแหน่ง และ canvas ที่ต้องการจะวาดลงไป และยังมี method ที่จำเป็นต่อเกม เช่น let Eat ที่เอาไว้ตรวจสอบการชนของ player และ foodsmall, foodbig เพื่อนำไปเพิ่มเป็น score นอกจากนี้ถ้า player ชน foodbig จะมีความสามารถ eat ghost ได้ด้วย นอกจากนั้นก็มี method move เอาไปให้ player เคลื่อนไหวตาม keyboard และ สุดท้าย method action จะทำการเรียกตัวเองซ้ำๆ เพื่อสร้าง animation ให้แก่ player

Class Ghost

- เป็นคลาสที่มี Method คล้ายกับ player ในส่วนที่แตกต่างกันนั้นคือ Ghost ไม่ได้ move ด้วย keyboard แต่จะ move ตามตำแหน่งที่กำหนดไว้ผ่าน Method findpath ซึ่งใช้ Algorithm Shortest path ของ Dijtra ในการเขียน ซึ่งในการเดินของ Ghost นั้นจะมี 2 mode คือ hunt กับ roaming ซึ่ง hunt นั้นจะทำการไล่ล่า player ตลอดเวลา ส่วน roaming จะทำการเดินแบบ random ทำให้สร้างสีสรรภายในเกมให้มีความหลากหลาย นอกจากนี้ยังมี run_mode และ deadmode ในส่วนของ run_mode นั้นเมื่อ player ทำการชน foodbig จะทำให้ ghost เกิด run_mode ขึ้นจะทำให้ ghost นั้นเดินช้าลงและสามารถถูกชนได้ในช่วงระยะเวลานึง ส่วน deadmode เมื่อ ghost อยู่ในสถานะ run_mode แล้วถูก player ชนจะทำให้เข้าสู่สถานะ deadmode ซึ่งในสถานะนี้ ghost จะเดินเร็วขึ้นและจะกลับไปยังจุดเกิดโดยผ่าน method findpath นั้นเอง

Class block

- จะทำหน้าที่แทนสิ่งกีดขวาง รูปสี่เหลี่ยมจัตุรัส ใช้สำหรับสร้าง map ขึ้นมาโดยจะมีการ กำหนดตำแหน่งในการสร้าง

Class foodsmall

- จะทำหน้าที่แทนอาหารของ player เมื่อ player มาชนจะได้ score ซึ่งในการสร้าง จะมีการกำหนดตำแหน่งเหมือนกับคลาสอื่นๆ

Class foodbig

- ทำหน้าที่เหมือนกับ foodsmall ทุกประการแต่จะแตกต่างกันเล็กน้อยคือจะมี animation การกระพริบเพิ่มเข้ามา

Class Point

- เป็นคลาสที่สร้างมาเพื่อ findpath โดยเฉพาะ ซึ่งจะมีตัวแปร x, y แทนตำแหน่ง isblock แทนว่าสามารถไปได้หรือไม่ isVisit คือแทนว่าได้ผ่านเส้นทางนี้ไปหรือยัง direction คือทิศทางที่ใช้ในการเคลื่อนที่ และสุดท้าย prev ไว้ชี้ point ก่อนหน้าเพื่อใช้ย้อนรอยเมื่อทำการหาเส้นทางที่สั้นที่สุดได้แล้ว
